Data Flow

Access Request Flow

1

Card Presented at Reader

User presents credential. BLU-IC2 hardware captures card data via OSDP/Wiegand protocol.

2

Thrall Receives from Hardware

Aether Thrall receives card data through Azure SDK wrapper.

3

AetherDB Credential Check

Thrall queries AetherDB (binary format) for card validity and permissions. Decision in <250ms.

4

Physical Action

If access granted, Thrall controls door strike, LED, and buzzer via SDK.

5

Event Logged to AetherDB

Access event written to 100K circular buffer in AetherDB.

6

Loom Synchronization

Loom translates event to SQL and syncs to Saga (PostgreSQL) and Bifrost (SQLite/PostgreSQL).

7

External Export

Bifrost exports event to Ambient.ai, SIEM, and central Lenel/OnGuard systems.
